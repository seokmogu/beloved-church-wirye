# 사랑하는교회 위례 웹사이트

Payload CMS 기반의 교회 운영형 웹사이트입니다. 목표는 일반 웹빌더처럼 자유 배치하는 것이 아니라, 교회 관리자가 관리자 화면에서 모든 공개 콘텐츠를 안정적으로 수정할 수 있는 전용 CMS 빌더를 제공하는 것입니다.

## 운영 원칙

- 하드코딩된 공개 콘텐츠를 두지 않습니다.
- 교회명, 로고, 예배 시간, 주소, 지도, SNS, YouTube, Footer 문구는 `사이트 설정` Global에서 관리합니다.
- 메뉴는 `Header` Global에서 관리하며, 코드에 고정 메뉴를 추가하지 않습니다.
- Footer 링크와 안내 문구는 `Footer` Global과 `사이트 설정`에서 관리합니다.
- 신규 페이지는 `Pages` 컬렉션에서 생성하고 블록으로 구성합니다.
- 홈 화면은 운영자가 섹션 노출 여부와 순서를 조정할 수 있어야 합니다.
- 공지사항, 주보, 설교, 새가족 등록은 각각 전용 컬렉션에서 관리합니다.
- YouTube/SNS 연동은 계정/채널 설정을 CMS에 두고, 코드는 자동 수집과 표시만 담당합니다.

## 관리자 주요 메뉴

- 사이트 설정: 교회 기본 정보, 홈 히어로, 예배 시간, 주소, 지도, SNS, 자동 연동 설정
- Header: 상단 메뉴 구성
- Footer: 하단 링크와 보조 안내
- Pages: 관리자가 새 페이지를 생성하고 블록으로 구성
- 공지사항: 교회 소식 관리
- 주보: 주보 PDF/이미지 업로드 및 공개 여부 관리
- 설교: 설교 영상 메타데이터 관리
- 새가족: 새가족 등록 내역 관리
- 특별 배너: 기간 한정 안내 배너 관리

## 로컬 관리자 계정

홈서버 개발 환경에는 로컬 관리자 계정이 준비되어 있습니다.

- URL: `http://100.94.135.5:3000/admin`
- 이메일: `admin@beloved.local`
- 비밀번호: `.admin.local.credentials` 파일에서 확인합니다.

`.admin.local.credentials`는 권한 `600`으로 생성되며 `.git/info/exclude`에 추가되어 있습니다. 비밀번호를 README, 이슈, 커밋, 배포 환경 문서에 복사하지 않습니다.

## 페이지 빌더 방향

관리자는 정해진 섹션 블록을 조립합니다.

- Hero
- Content
- Media
- YouTube
- Instagram
- Schedule
- Google Drive
- Offering
- Announcements
- Bulletins

자유 드래그 빌더가 아니라 검증된 섹션을 조합하는 방식으로 유지합니다. 이렇게 해야 모바일, 접근성, 브랜드 일관성, 관리 난이도를 안정적으로 유지할 수 있습니다.

## 완결 기준

사이트를 운영 가능 상태로 판단하려면 다음이 충족되어야 합니다.

- 빌드와 타입 체크가 통과합니다.
- 관리자 화면에서 Header/Footer/Home/About/Worship에 보이는 모든 문구와 링크를 수정할 수 있습니다.
- 신규 페이지를 CMS에서 생성하고 메뉴에 연결할 수 있습니다.
- 공지사항, 주보, 설교가 CMS 데이터로 표시됩니다.
- YouTube 채널과 Instagram 계정/게시물 설정이 코드가 아닌 CMS에 있습니다.
- 오픈 전 필수 입력값 누락을 관리자가 확인할 수 있습니다.

## 디자인 기준

- 홈 히어로, 소개, 설교, Instagram, 위치 섹션은 CMS 데이터를 기반으로 렌더링합니다.
- `사이트 설정 > 디자인`에서 메인 컬러, 강조 컬러, 배경 컬러, Header/Footer 컬러, 히어로 오버레이, 페이지/어두운 섹션 배경 이미지를 관리합니다.
- 디자인 탭의 `디자인 미리보기`는 저장 전 색상 입력값을 즉시 반영합니다.
- 배경 이미지는 업로드 필드 특성상 저장 후 실제 사이트 미리보기로 확인합니다.
- CMS에 히어로 이미지나 지도 API 키가 없어도 빈 박스나 개발자용 메시지가 보이지 않아야 합니다.
- 관리자가 섹션 노출 여부와 순서를 변경해도 모바일/데스크톱 레이아웃이 깨지지 않아야 합니다.
- 실제 운영 전에는 CMS Media에 교회 실사진을 등록해 히어로 이미지로 사용하는 것을 권장합니다.

## 개발 명령

```bash
pnpm dev
pnpm build
pnpm generate:types
pnpm generate:importmap
pnpm test:int
pnpm test:e2e
```

현재 로컬 빌드는 DB 연결이 필요합니다. 홈서버에서 `POSTGRES_URL`이 로컬 Postgres를 가리키는 경우 Postgres가 실행 중이어야 합니다.
