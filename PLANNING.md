### 현재 상태 분석
- **구조**: Next.js + Payload CMS 기반, 깔끔한 단일 페이지 레이아웃
- **콘텐츠**: 교회 소개, 예배 안내, 공지사항, 설교 영상, 인스타그램 피드
- **강점**: 
  - 모던한 디자인과 가독성
  - YouTube 설교 영상 자동 연동
  - Instagram 피드 통합
  - 반응형 레이아웃
- **개선 필요 영역**: 새가족 온라인 등록, 헌금 안내, 소그룹 정보, 접근성

## 개선 제안 (2026-03-31)

### 1. 새가족 온라인 등록 시스템 🔧 **코드 작업 필요**
**문제점**: 
- 현재 새가족이 처음 방문 시 등록 방법이 없음
- 오프라인에서만 등록 가능한 구조

**제안**:
- `/newcomer` 페이지 신규 생성
- Payload CMS에 `Newcomers` 컬렉션 추가 (이름, 연락처, 가정 상황, 관심 사역, 방문 경로 등)
- 간단한 폼으로 정보 수집 → 새가족부 담당자에게 이메일 알림
- 메인 페이지 CTA(Call To Action) 버튼 추가: "새가족 등록하기"

**CMS 가능 여부**: ❌ (컬렉션/페이지 개발 필요)
**우선순위**: ⭐⭐⭐ 높음

---

### 2. 헌금 안내 페이지 ✅ **CMS로 충분**
**문제점**:
- 헌금 계좌/송금 방법이 명시되지 않음
- 새신자나 온라인 참여자가 헌금 방법을 몰라 어려움

**제안**:
- Payload Global Settings에 `Offering` 항목 추가:
  - 계좌번호
  - 은행명
  - 예금주
  - 카카오페이 QR (선택)
- 메인 페이지 footer나 예배 섹션에 "헌금 안내" 링크
- 모달 또는 독립 페이지로 구현

**CMS 가능 여부**: ✅ (Global 설정 + 간단한 페이지 컴포넌트)
**우선순위**: ⭐⭐⭐ 높음

---

### 3. 소그룹/사역 소개 페이지 ✅ **CMS로 충분**
**문제점**:
- "소그룹과 다양한 사역으로 서로 연결된 공동체"라고 소개하지만 구체적 정보 없음
- 참여 방법을 알 수 없음

**제안**:
- Payload에 `Ministries` 컬렉션 추가 (사역명, 설명, 담당자, 일정, 대표 이미지)
- `/ministries` 페이지 생성 또는 메인 페이지에 섹션 추가
- 각 사역별 신청 버튼(연락처 남기기 또는 이메일)

**CMS 가능 여부**: ✅ (컬렉션 생성 + 템플릿)
**우선순위**: ⭐⭐ 중간

---

### 4. 주보(Bulletin) 다운로드 접근성 개선 🔧 **코드 작업 필요**
**문제점**:
- 메인 네비게이션에 "주보" 링크는 있으나(`/bulletins`) 현재 상태 확인 불가
- PDF 다운로드가 직관적이지 않을 가능성

**제안**:
- `/bulletins` 페이지에 최신 주보를 큰 썸네일로 표시
- "이번 주 주보 다운로드" 버튼을 눈에 잘 띄게 배치
- 과거 주보 아카이브 기능 (날짜별 필터링)
- Payload `Bulletins` 컬렉션에 PDF 업로드 + 날짜 메타데이터

**CMS 가능 여부**: 🔶 (CMS 컬렉션은 가능하나 UI 개선은 코드 필요)
**우선순위**: ⭐⭐ 중간

---

### 5. 웹 접근성 및 SEO 개선 🔧 **코드 작업 필요**
**문제점**:
- 검색 엔진 최적화(SEO) 메타 태그 확인 필요
- 스크린 리더 사용자 고려한 ARIA 레이블 미비

**제안**:
- `<meta>` 태그: description, og:image, Twitter Card 등 추가
- 시맨틱 HTML 개선 (예: `<article>`, `<section>` 적절히 활용)
- 이미지 alt 텍스트 필수화 (Payload Media 컬렉션에 alt 필드 강제)
- Google Search Console, Naver 웹마스터도구 등록

**CMS 가능 여부**: 🔶 (일부 Payload SEO 플러그인으로 가능, 코드 수정도 필요)
**우선순위**: ⭐ 낮음 (장기 개선)
