### 🔍 한국 교회 웹사이트 모범 사례 (2026년 벤치마킹)

웹 검색을 통해 파악한 한국 및 글로벌 교회 웹사이트 트렌드:

#### 핵심 트렌드 5가지
1. **포용적 접근성** - 모든 연령대와 장애 유무에 관계없이 접근 가능
2. **모바일 우선 설계** - 대부분의 방문이 모바일에서 발생
3. **온라인 헌금 시스템** - 계좌 정보 또는 간편결제 필수
4. **새가족 환영 프로세스** - 온라인 등록 폼으로 허들 낮추기 ✅ (이미 구현)
5. **사역/소그룹 상세 안내** - 참여 유도를 위한 구체적 정보 제공

### 🎯 개선 제안 (2026-04-01 기준)

#### 🔴 최우선 (CMS 작업 즉시 가능)

##### 1. 주보 PDF 업로드 ✅ **CMS만으로 가능**
**현재 상태**: `/bulletins` 페이지에 "등록된 주보가 없습니다" 표시  
**문제점**: 
- 메인 네비게이션에 주보 링크는 있으나 실제 콘텐츠 없음
- 주보는 교회의 핵심 콘텐츠

**제안**:
- Payload CMS `Bulletins` 컬렉션에 PDF 파일 업로드
- 최소 최근 3-4주치 주보 등록
- 날짜별 정렬로 최신 주보가 상단에 표시되도록

**CMS 가능 여부**: ✅  
**우선순위**: ⭐⭐⭐⭐⭐  
**담당**: 교회 웹 관리자

---

##### 2. 헌금 안내 추가 🔶 **CMS + 간단한 코드**
**현재 상태**: 헌금 관련 정보 전무  
**문제점**: 
- 온라인 참여자나 새신자가 헌금 방법을 알 수 없음
- 투명성과 접근성 향상 필요

**제안**:
- **1단계 (CMS)**: Payload Global Settings에 헌금 정보 추가
  - 계좌번호, 은행명, 예금주
  - 카카오페이 QR 코드 (선택)
- **2단계 (코드)**: `/offering` 페이지 생성 또는 footer 링크 추가

**CMS 가능 여부**: 🔶 (CMS로 데이터 관리, 페이지는 코드 필요)  
**우선순위**: ⭐⭐⭐⭐  
**담당**: 기획 에이전트 (Global 설정) + 빌더 에이전트 (페이지)

---

#### 🟠 높음 (코드 작업 필요, 1-2주 내)

##### 3. 설교 메타데이터 보강 🔶 **CMS + 코드**
**현재 상태**: YouTube 자동 제목만 ("Beloved Church 26.XX.XX. 주일예배")  
**문제점**:
- 설교 본문 제목 없음 (예: "사랑의 완성", "십자가의 능력")
- 성경 본문 정보 없음 (예: 요한복음 3:16-21)
- 설교자 정보 없음
- SEO 불리

**제안**:
- **CMS**: 각 설교 영상에 메타데이터 추가
  - 설교 제목
  - 성경 본문
  - 설교자 (차원석 목사)
  - 설교 요약 (2-3줄)
- **코드**: 설교 카드 UI 개선 (제목+본문 표시)

**CMS 가능 여부**: 🔶  
**우선순위**: ⭐⭐⭐  
**담당**: 웹 관리자 (메타데이터) + 빌더 (UI)

---

#### 🟡 중간 (2-4주 내)

##### 4. 사역/소그룹 소개 페이지 🔶 **CMS + 코드**
**현재 상태**: 교회 소개에서 "소그룹과 사역"이라고만 언급  
**문제점**:
- 구체적인 사역 정보 없음
- 새가족 폼에 관심 사역 선택지는 있으나 실제 페이지 없음

**제안**:
- **CMS**: `Ministries` 컬렉션 추가
  - 사역명, 설명, 일정, 담당자, 대표 이미지
  - 예: 청년부, 찬양팀, 성경공부, 봉사, 기도회, 소그룹
- **코드**: `/ministries` 페이지 생성
  - 사역별 카드 레이아웃
  - 참여 신청 버튼

**CMS 가능 여부**: 🔶  
**우선순위**: ⭐⭐  
**담당**: 기획 (컬렉션 설계) + 빌더 (페이지)
